# Demo Script – Leantime Testing (10–15 Minutes)

## 1. Introduction (1–2 min)
- Introduce the team and the Track B objective.
- Brief description of Leantime and what we tested.

## 2. Environment Overview (2–3 min)
- Show containers running (Leantime + DB).
- Confirm base URL and health (smoke test).

## 3. Functional Tests (3–4 min)
- Demonstrate authentication and RBAC tests (F1, F2).
- Show CRUD and search/filter/pagination tests (F3–F5).
- Highlight how evidence is captured (logs, screenshots, DB snapshots).

## 4. Non-Functional Tests (3–4 min)
- Present performance tests (P1 baseline, P2 stress) with charts.
- Show security scan summary (S1) and injection testing (S3).
- Mention usability/accessibility findings (U1).

## 5. Findings, Risks, and Recommendations (2–3 min)
- Summarize key defects and risks.
- Present top recommendations and expected impact.

## 6. Closing (1 min)
- State whether quality gates were met.
- Mention possible next steps or future improvements.
